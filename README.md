# 🀄 ໄພ່ລາວ — Lao Multiplayer Card Table

ເວັບແອັບເກມໄພ່ (ຄ້າຍ Tiến Lên / ໄພ່ແຄງ) ຫຼິ້ນໄດ້ 2–5 ຄົນຜ່ານມືຖື/ຄອມ,
ສ້າງດ້ວຍ HTML + CSS + JavaScript ລ້ວນໆ (ບໍ່ຕ້ອງ build) ແລະ ໃຊ້
**Firebase Realtime Database** (ຟຣີ) ເປັນ "ເຊີບເວີ" ສຳລັບຊິງຄ໌ຂໍ້ມູນລະຫວ່າງຜູ້ຫຼິ້ນ
ເພື່ອໃຫ້ສາມາດ deploy ຂຶ້ນ **GitHub Pages** ໄດ້ໂດຍກົງ (GitHub Pages ຮອງຮັບແຕ່ໄຟລ໌ static).

---

## 1. ໂຄງສ້າງໄຟລ໌

```
card-game/
├── index.html     ໜ້າຕົ້ນສະບັບ (Landing / Waiting room / Game table)
├── style.css      ໜ້າຕາ ໂຕະໄພ່, ໄພ່, ອານິເມຊັນ
├── app.js         ກົດກະຕິກາເກມ + ການເຊື່ອມຕໍ່ Firebase
├── config.js      ⚠️ ໃສ່ຄ່າ Firebase ຂອງທ່ານທີ່ນີ້ (ຂັ້ນຕອນລຸ່ມນີ້)
└── README.md      ໄຟລ໌ນີ້
```

## 2. ຕັ້ງຄ່າ Firebase (ຟຣີ, ໃຊ້ເວລາ ~5 ນາທີ)

1. ໄປທີ່ <https://console.firebase.google.com/> → **Add project** → ຕັ້ງຊື່ຫຍັງກໍໄດ້ (ບໍ່ຈຳເປັນຕ້ອງເປີດ Google Analytics)
2. ໃນເມນູຊ້າຍ: **Build → Realtime Database → Create Database**
   - ເລືອກ location ໃກ້ທ່ານທີ່ສຸດ (ເຊັ່ນ Singapore)
   - ເລືອກ **Start in test mode** (ອະນຸຍາດອ່ານ/ຂຽນຊົ່ວຄາວ 30 ວັນ — ພໍສຳລັບຫຼິ້ນກັບໝູ່)
3. ໄປທີ່ **Project settings** (ຮູບເຟືອງມຸມຊ້າຍເທິງ) → ແທັບ **General** → ເລື່ອນລົງ **Your apps**
   → ກົດໄອຄອນ **Web `</>`** → ຕັ້ງຊື່ແອັບ → ຈະໄດ້ Object `firebaseConfig`
4. ກັອບປີ້ຄ່າທັງໝົດໄປວາງໃນ `config.js` (ແທນທີ່ `PASTE_YOUR_...` ທຸກອັນ)
5. (ແນະນຳ) ຫຼັງຈາກທົດລອງໃຊ້ໄດ້ແລ້ວ, ໄປແກ້ **Realtime Database → Rules** ໃຫ້ຮັດກຸມຂຶ້ນເລັກນ້ອຍ ເຊັ່ນ:
   ```json
   {
     "rules": {
       "rooms": {
         "$roomId": {
           ".read": true,
           ".write": true
         }
       }
     }
   }
   ```
   > ⚠️ **ໝາຍເຫດຄວາມປອດໄພ:** ໃນໂໝດ test/rule ແບບເປີດນີ້ ຜູ້ຫຼິ້ນທີ່ຮູ້ເລີກໆ (ເປີດ DevTools)
   > ຈະສາມາດອ່ານໄພ່ໃນມືຄົນອື່ນຈາກຖານຂໍ້ມູນໄດ້ໂດຍກົງ (ໜ້າຈໍປົກກະຕິບໍ່ສະແດງ). ນີ້ແມ່ນຂໍ້ຈຳກັດ
   > ຂອງການເຮັດ "serverless" ເທິງ GitHub Pages ໂດຍບໍ່ມີ backend server ແທ້ໆ — ພໍເໝາະສຳລັບ
   > ຫຼິ້ນສະໜຸກກັບໝູ່ (casual play), ບໍ່ແມ່ນສຳລັບການແຂ່ງຂັນທີ່ຕ້ອງກັນໂກງແທ້ຈິງ.

## 3. ລອງໃນເຄື່ອງກ່ອນ (Local test)

ເປີດໄຟລ໌ `index.html` ຜ່ານ local server ງ່າຍໆ (ບໍ່ໃຊ້ double-click ໂດຍກົງ ເພາະບາງ browser
ຈະບລັອກ fetch ຂອງໄຟລ໌ local):

```bash
cd card-game
python3 -m http.server 8080
# ຫຼື: npx serve .
```

ແລ້ວເປີດ `http://localhost:8080` ໃນຫຼາຍແທັບ/ຫຼາຍອຸປະກອນ ເພື່ອທົດລອງຫຼາຍຄົນ.

## 4. Deploy ຂຶ້ນ GitHub Pages

```bash
git init
git add .
git commit -m "Lao card table"
git branch -M main
git remote add origin https://github.com/<ຊື່ຜູ້ໃຊ້>/<ຊື່repo>.git
git push -u origin main
```

ຈາກນັ້ນໃນໜ້າ GitHub ຂອງ repo: **Settings → Pages → Build and deployment**
→ Source: `Deploy from a branch` → Branch: `main` / `root` → **Save**.
ລໍຖ້າ ~1 ນາທີ ຈະໄດ້ລິ້ງແບບ `https://<ຊື່ຜູ້ໃຊ້>.github.io/<ຊື່repo>/` — ແບ່ງປັນລິ້ງນີ້ ແລະ
ລະຫັດຫ້ອງ 4 ຕົວເລກໃຫ້ໝູ່ໄດ້ເລີຍ.

---

## 5. ວິທີຫຼິ້ນ

1. ຄົນທຳອິດເລືອກ **"ສ້າງຫ້ອງ"** → ເລືອກຈຳນວນຄົນ (2–5) → ຕັ້ງລະຫັດ 4 ຕົວເລກ
2. ຄົນອື່ນເລືອກ **"ເຂົ້າຮ່ວມ"** → ເລືອກຈຳນວນຄົນໃຫ້ກົງກັນ → ໃສ່ລະຫັດດຽວກັນ
3. ເມື່ອຄົນຄົບ ເຈົ້າຂອງຫ້ອງ (ຄົນສ້າງ) ກົດ **"ເລີ່ມເກມ"**
4. ແຈກຄົນລະ 9 ໃບ, ໃຜກົດລົງໄພ່ກ່ອນເປັນຄົນນຳຮອບ, ຈາກນັ້ນວົນຂວາມືອັດຕະໂນມັດ
5. ແຕະໄພ່ໃນມືເພື່ອເລືອກ (ຫຼາຍໃບໄດ້) → ກົດ **"ລົງໄພ່"** — ຖ້າແຕະໃບດຽວແລ້ວມີຫຼາຍທາງລົງໄດ້
   (ເຊັ່ນ ໃບດ່ຽວ/ຄູ່/ເປົາ/ລຽງ) ລະບົບຈະເດັ້ງ popup ໃຫ້ເລືອກ
6. ໝົດ 20 ວິນາທີແລ້ວບໍ່ລົງ → ລະບົບຈະ **ຜ່ານ (pass)** ໃຫ້ອັດຕະໂນມັດ (ຖ້າເປັນຄົນນຳຮອບ ຈະລົງໃບນ້ອຍສຸດ
   ໃຫ້ອັດຕະໂນມັດ)
7. ໝົດໄພ່ໃນມືກ່ອນ = ຊະນະ! ກົດ **"ຫຼິ້ນໃໝ່"** ເພື່ອແຈກຮອບໃໝ່ (ຄົນເກົ່າ, ຫ້ອງເກົ່າ)

## 6. ກົດກະຕິກາທີ່ຕັ້ງໄວ້ໃນລະບົບ (ແກ້ໄດ້ໃນ `app.js` → `CONFIG`)

| ກົດ | ຄ່າເລີ່ມຕົ້ນ |
| --- | --- |
| ຈຳນວນໄພ່ຕໍ່ຄົນ | 9 ໃບ |
| ເວລາຄິດຕໍ່ຮອບ | 20 ວິນາທີ |
| ໄພ່ຄູ່ (Pair) | ເລກດຽວກັນ + ສີດຽວກັນ (ດຳ/ດຳ ຫຼື ແດງ/ແດງ) |
| ໄພ່ເປົາ/ຕອງ (Triple) | ເລກດຽວກັນ 3 ໃບ — ສາມາດທັບໄພ່ດ່ຽວ 10/J/Q/K ໄດ້ |
| ໄພ່ລຽງ (Straight) | ລຽງ 3+ ໃບ ດອກດຽວກັນ, ບໍ່ນັບໃບ 2 — ສາມາດທັບໄພ່ດ່ຽວໃດໆ (ນອກຈາກ 2) ໄດ້ |
| ລຽງພິເສດ (Super Straight) | ລຽງ 4+ ໃບ ດອກດຽວກັນ — ໃຊ້ທັບໄພ່ດ່ຽວເລກ 2 ເທົ່ານັ້ນ, ຊະນະທັນທີເມື່ອທັບໄດ້ |
| ເປົາພິເສດ (Super Quad) | 4 ໃບເລກດຽວກັນຢູ່ໃນມືຕອນແຈກ — ຊະນະທັນທີ |

> ໝາຍເຫດ: ຄຳອະທິບາຍກົດກະຕິກາຕົ້ນສະບັບບາງຈຸດ (ເຊັ່ນ ການທັບໄພ່ນ້ອຍດ້ວຍໄພ່ລຽງ) ອາດຕີຄວາມໄດ້
> ຫຼາຍແບບ — ຂ້ອຍໄດ້ເລືອກຕີຄວາມແບບໜຶ່ງທີ່ຫຼິ້ນໄດ້ຄົບຖ້ວນ ແລະ ບໍ່ຄ້າງເກມ, ຖ້າຢາກປັບໃຫ້ກົງກັບກົດ
> ຂອງກຸ່ມທ່ານແທ້ໆ, ແກ້ໄດ້ໃນ `CONFIG` ແລະ ຟັງຊັນ `canBeat()` ໃນ `app.js`.

## 7. ຂໍ້ຈຳກັດທີ່ຄວນຮູ້ (ຍ້ອນເປັນ static site ບໍ່ມີ backend server ແທ້)

- **ຄົນສ້າງຫ້ອງ (host)** ຕ້ອງເປີດແທັບໄວ້ຕະຫຼອດເກມ — ນາທີຈັບເວລາ ແລະ ການກວດກົດກະຕິກາ
  ແມ່ນຄຳນວນຢູ່ໃນເຄື່ອງຂອງຄົນສ້າງຫ້ອງ. ຖ້າຄົນສ້າງຫ້ອງອອກຈາກແທັບ ໂມງຈະຢຸດແລ່ນ (ຄົນອື່ນຍັງລົງໄພ່
  ໄດ້ປົກກະຕິ, ພຽງແຕ່ບໍ່ມີ auto-pass ເມື່ອໝົດເວລາ)
- ຄວາມເປັນສ່ວນຕົວຂອງໄພ່ໃນມືອີງໃສ່ໜ້າຈໍເທົ່ານັ້ນ (ເບິ່ງໝາຍເຫດຄວາມປອດໄພຂ້າງເທິງ)
- ອານິເມຊັນເປັນແບບ 2D CSS (ບໍ່ແມ່ນ 3D Pixar ແບບ Unity ຕົ້ນສະບັບ) — ຖ້າຕ້ອງການ 3D ແທ້ຈິງ
  ຄວນພັດທະນາເປັນແອັບ Unity ແຍກຕ່າງຫາກຕາມ GDD ທີ່ວາງໄວ້, ເວັບແອັບນີ້ແມ່ນເວີຊັນທີ່ຫຼິ້ນໄດ້ໄວ
  ຜ່ານ browser ໂດຍບໍ່ຕ້ອງຕິດຕັ້ງຫຍັງ
- ບໍ່ມີລະບົບບັນຊີຜູ້ໃຊ້/login — ຊື່ ແລະ "ຕົວຕົນ" ຂອງທ່ານຖືກຈື່ໄວ້ໃນ browser (localStorage)
  ຂອງອຸປະກອນນັ້ນເທົ່ານັ້ນ

## 8. ຢາກຕໍ່ຍອດ?

ຈຸດທີ່ດີສຳລັບຂະຫຍາຍ: ອັນດັບຜູ້ຫຼິ້ນ (ບໍ່ແມ່ນແຕ່ຜູ້ຊະນະຄົນດຽວ), ຫ້ອງລໍຖ້າແບບມີ chat,
Cloud Functions ສຳລັບເຊັກກົດກະຕິກາຝັ່ງເຊີບເວີແທ້ (ປ້ອງກັນການໂກງ 100%), ຫຼື ປ່ຽນ UI
ໃຫ້ເປັນ 3D ດ້ວຍ Three.js.
